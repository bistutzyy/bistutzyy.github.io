Claude Code 手机版实战：多轮修正 Word 文档直到完美提交

发表于2026-05-14|更新于2026-05-14|AI编程
|总字数:879|阅读时长:3分钟|浏览量:

场景

今天在手机上完成了一次完整的文档修改工作流：一份数据库实验报告需要做格式规范化，通过 Claude Code 直接在手机上操作 .docx 文件，经过两轮迭代修改，最终达到提交标准。

整个过程全部在手机上完成：接收文件 → 解压 XML → Claude 分析修改 → 打包 → 发送回微信。全程没有打开电脑上的 Word，甚至没有离开聊天界面。

任务背景

手头有份《数据库原理及应用 实验二》的实验报告，上一轮已经改过主要问题（NVARCHAR、N 前缀、触发器集合写法、权限管理），但还有几处格式细节需要打磨：

3 第（四）1 安全管理 CREATE LOGIN 后缺 GO
4 第（四）3 授权 GRANT 前缺 USE/GO

技术原理

很多人不知道：.docx 文件本质上是一个 ZIP 压缩包，里面是一堆 XML 文件。所以可以直接解压 → 改 XML → 打包回去，完全不需要 Word。

试想一下这个流程：老师在微信上发来一份实验报告模板，你在手机上用 Claude Code 直接打开、修改、打包、发回——整个过程不超过十分钟。不需要传到电脑、不需要打开 Office、不需要来回切换设备。这就是 AI CLI 工具在移动端的真正价值所在。

适用场景： 实验报告、课程作业、社团文档、实习周报——凡是格式规范化的 .docx 文档修改，都可以用这套流程解决。
