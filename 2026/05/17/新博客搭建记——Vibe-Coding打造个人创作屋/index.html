新博客搭建记——用 Vibe Coding 打造「桃之夭夭の创作屋」

发表于2026-05-17|更新于2026-05-17|项目
|总字数:1.7k|阅读时长:5分钟|浏览量:

今天花了一整天，用 Vibe Coding 的方式从零搭建了一个全新的个人主页——桃之夭夭の创作屋。

地址：https://tzyy11.vercel.app

这篇文章记录整个过程：为什么要建新站、三款 AI 工具如何分工协作、功能设计上的巧思，以及部署上线后的效果展示。

为什么要建新站

在这之前，我已经有两个博客了：

  1. bistutzyy.github.io — Hexo + Butterfly 主题，写了近两年
  2. blog1-reimu.vercel.app — Hexo + Reimu 主题，年初刚搭的

两个博客各有特色，但问题也很明显：内容分散、管理碎片化、主题定制受限。每次想改个样式都要折腾半天配置，Butterfly 标签和 Reimu 标签还不互通，迁移文章特别痛苦。

核心诉求其实很简单：有一个完全由自己掌控的空间——不为搜索引擎优化，不为流量变现，纯粹用来记录和展示。

所以今天，我决定用 AI 工具从零搭建第三个站——一个完全自由的个人创作屋。

搭建过程：三款 AI 工具的接力赛

整个搭建过程像一个接力赛，三款 AI 工具各司其职：

OpenAI Codex — 主页整体设计

Codex 负责了主页的视觉框架。我给它的需求是”设计一个带侧边栏的个人主页，日系插画风格，清新可爱”。

它生成了完整的 index.html：

  • 左侧导航栏（首页/归档/文章/说说/相册/友链/留言墙）
  • 中间大标题 “Hello” 渐变色 + 手写体
  • 双语标语 “有趣的灵魂终会遇见”
  • 右侧动漫少女角色立绘（金发蓝眼，蝴蝶结）
  • 三只漂浮小猫插图（画画的、坐在花旁的、抱书读的）
  • 版权信息 + 社交图标

Codex 对视觉风格的理解很到位——柔和的浅色调、圆角设计、合适的留白，整个首页没有写一行 CSS 我就拿到了成品。

Cursor — 碎片化页面设计

Cursor 负责了独立页面的设计：

  • 相册页：卡片式布局，点击进入子页面查看照片网格
  • 友链页：本站信息 + 友链卡片，带说明和申请指南
  • 留言墙：渐变色标题、表单三栏布局、留言卡片带糖果色头像和系统标签
  • 归档页：年份时间线，文章按日期排列

Cursor 的优势在于碎片化任务——给它一个具体页面的需求，它能快速生成符合整体风格的代码。

Claude Code + DeepSeek V4 Pro — 文章体系与全局调优

Claude Code 负责了最核心的内容系统：

  1. build_content.py 构建脚本：从 Markdown 文章 + frontmatter 生成文章列表、详情页、说说页、归档时间线
  2. 21 篇文章迁移：从 Hexo 博客的 Butterfly 标签语法全部转为标准 Markdown，保留核心内容和配图
  3. CSS 全局调优：macOS 终端风格代码块、艺术字语录、表格样式、移动端适配、侧边栏折叠
  4. 双语支持：每篇文章中英双语，页面级语言切换

整个过程中，我在微信上给 Claude Code 发需求，它分析、生成、我审查修改，全程没有打开传统 IDE。

功能设计亮点

1. 首页

首页是整个创作屋的门面——Codex 设计的动漫少女插画 + 手绘小猫 + 渐变色标题，营造出”有趣的灵魂终会遇见”的氛围。

2. 文章系统

文章列表用卡片式布局，每篇文章显示标题、摘要、日期、字数、阅读量、预计阅读时间。支持中英双语切换，代码块用了 macOS 终端风格——暗色背景 + 顶部标题栏 + 三色圆点 + 语法高亮。

21 篇文章从 Hexo 博客完整迁移，Butterfly 标签语法全部转为标准 Markdown，去掉了封面图和装饰效果，保留核心内容和配图。新增了《关于桃之夭夭の创作屋》这篇介绍文。

3. 说说 — 随机艺术效果

这个是重点——说说页面里的每条卡片 样式都是随机的。

不是固定的模板套用，而是每次构建时根据内容自动分配不同的视觉效果：

样式有哪些
  • 极光渐变背景 + 描边
  • 樱花粉色调
  • 浅蓝渐变 + 水波纹理
  • 车票风格（虚线边框 + TICKET 水印）
  • 信封样式
  • 磨砂玻璃效果
  • 复古便签
  • 糖果条纹
  • 霓虹描边
  • 日系手账风
  • 水彩晕染

每种样式都有独特的颜色、阴影、装饰元素和字体效果。

这些样式的设计灵感来自原 Hexo 博客的 Butterfly 标签体系——特别是 Akilar 的糖果屋风格，擅长用 CSS 创造富有情感色彩的文字视觉效果。

4. 其他功能

  • 相册：三个相册卡片，第一个收录了 30 张动漫图片
  • 侧边栏折叠：右上角按钮一键收起，只留图标导航
  • 移动端适配：768px / 480px 两档响应式，表格横向滚动，代码块缩小
  • 代码高亮：highlight.js 自动着色，支持多种语言
  • 天气特效：右下角可切换雨天模式

与旧博客的关系

原 Hexo 博客 bistutzyy.github.io 保持不变，新创作屋独立部署在 Vercel。

两者的定位不同：

旧博客 — Hexo 框架 + Butterfly 主题，文章同步更新，作为传统博客的保留地。

新创作屋 — 纯静态 HTML + Python 构建，更自由、更轻量、完全由自己定制。

后续计划

文章迁移 — 21 篇文章已从 Hexo 博客完整迁移

说说迁移 — 5 条说说已迁移，带随机艺术效果

相册搭建 — 三个相册已创建，第一个收录动漫图片

更多相册 — 生活碎片、旅行街景待填充

图片上传 — markdown 直接引 COS 链接

总结

一天时间，三个 AI 工具分工协作，从零搭建了一个完整的个人创作屋。

最让我惊喜的是：Vibe Coding 让”建站”这件事变得前所未有的自由。不需要纠结框架选型、不需要手写大量重复 CSS、不需要在配置文件里迷失——我只需要描述我想要什么，AI 助手们各司其职把它实现出来。

Codex 负责视觉框架，Cursor 处理碎片页面，Claude Code 管理内容体系——每个工具都在自己最擅长的领域发挥作用。

新站地址：https://tzyy11.vercel.app

欢迎来逛。
