71 个品牌设计系统一键注入——awesome-design-md 做成 Claude Code Skill

发表于2026-05-18|更新于2026-05-18|开发工具
|总字数:1.2k|阅读时长:4分钟|浏览量:

项目地址：github.com/VoltAgent/awesome-design-md — 72,000+ Stars，MIT 许可证

这是什么？

awesome-design-md 是 VoltAgent 团队维护的开源仓库，把 71 个知名产品 的设计系统（配色、字体、间距、组件样式、阴影、响应式规则）提炼成 AI 能直接阅读的 DESIGN.md 文件。

你选一个品牌，把它的 DESIGN.md 放进项目根目录，然后对 Claude Code 说”请根据 DESIGN.md 的设计规范生成界面”——AI 就会按那个品牌的风格来写代码。

DESIGN.md 的结构——9 大模块

每个品牌的 DESIGN.md 都遵循 Google Stitch 引入的标准格式，包含 9 个章节：

9 大模块详解
#模块内容示例
1Visual Theme & Atmosphere“极简高对比度，冷色调技术感”
2Color Palette & Roles--accent: #5E6AD2 --bg-primary: #000000 语义色板
3Typography Rules字体家族 + Display/H1/H2/Body/Caption 字号层级表
4Component Stylings按钮/卡片/输入框 的 default/hover/active/disabled/focus 五态样式
5Layout Principles间距比例尺 4/8/12/16/24/32/48/64px，12 列网格
6Depth & Elevation0 级（平面）→ 3 级（模态弹窗）四层阴影系统
7Do’s and Don’ts“✅ 用语义色名，不要直接写 hex” 之类设计护栏
8Responsive Behavior断点 <640 / 640-1024 / >1024，触控目标 ≥44px
9Agent Prompt Guide快速颜色参考 + 现成 prompt 模板

把它做成了 Claude Code Skill

为了不用每次都去 GitHub 翻仓库，我把它封装成了一个 Claude Code Skill，存储在本地技能库中。

Skill 做了什么

Skill 文件：C:\Users\lenovo\.claude\skills\design-md\SKILL.md（183 行），已自动加载到 Claude Code 的技能列表。

核心能力：

  1. 品牌检索 — 71 个品牌按 9 大类组织，可以说”给我 Tesla 风格”直接定位
  2. 一键应用 — 从 GitHub clone 仓库 → 复制 DESIGN.md 到项目根目录
  3. 设计护栏 — 自动遵循 DESIGN.md 中的 9 大模块规则生成 UI，不跑偏
  4. 视觉预览 — 每个品牌附带 preview.html + preview-dark.html，可在浏览器中直接看色板、字号比例、按钮样式

实战演示

场景： 一个个人主页项目，想要 Linear 那种极简精准的审美风格。

Claude 会读取 Linear 的设计规范：

  • 配色 — 极简黑白，紫色点缀 #5E6AD2
  • 字体 — Inter Weight 510，紧凑字间距
  • 组件 — 微妙的边框、精确的圆角、克制的阴影层级
  • 布局 — 精准对齐、清晰信息层级

然后输出符合 Linear 风格的代码。

为什么值得用

没有 DESIGN.md 时，AI 对”好看”的理解是模糊的——你只能说”做大一点””颜色亮一点”这种抽象的反馈。有了 DESIGN.md，AI 有了具体的数值参考：字号多少 px、色值是什么 hex、间距用哪个 scale 格、阴影用哪个层级。

核心收益：

之前 之后
“这个卡片好看点” “按 DESIGN.md §4 Component Stylings 中的 Card 规范”
“字体大一点” “按 DESIGN.md §3 Typography 中的 H2 级别”
“颜色不对” “按 DESIGN.md §2 Color Palette 中的 accent 色值”
每次重新描述设计 DESIGN.md 一次写入，全局生效
