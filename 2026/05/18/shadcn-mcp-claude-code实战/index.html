Claude Code 新 MCP 实战——用 shadcn/ui MCP 给 React 项目加组件

发表于2026-05-18|更新于2026-05-18|开发工具
|总字数:1.2k|阅读时长:4分钟|浏览量:

MCP（Model Context Protocol）是一种让 AI 助手与外部工具沟通的协议。Claude Code 通过 MCP 可以直接调用外部服务——比如 shadcn/ui 组件库。

最近 Claude Code 集成了 React Bits 提供的 MCP 服务器，可以直接在对话中用 AI 搜索、浏览、添加 shadcn/ui 组件。

React Bits 是什么？

React Bits 是一个 React 组件资源库，收录了大量 shadcn/ui 组件、动画、示例代码等。它的 MCP 服务让 Claude Code 能直接在命令行里操作 shadcn 组件——搜索、预览、安装，全程不用离开终端。

MCP 配置地址：https://reactbits.dev/get-started/mcp

MCP 工具一览

Claude Code 接入 React Bits MCP 后，提供了 7 个工具：

MCP 工具清单
工具名功能
get_project_registries查看当前项目配置了哪些组件注册表
list_items_in_registries列出注册表中所有可用的组件（shadcn 有 405 个）
search_items_in_registries模糊搜索组件名
view_items_in_registries查看组件详情（类型、文件数、依赖）
get_item_examples_from_registries获取组件的完整示例代码
get_add_command_for_items生成添加组件的 CLI 命令
get_audit_checklist添加组件后的检查清单

实战：给个人主页加 shadcn 组件

以一个 React + TypeScript + Vite 的个人主页项目为例，演示完整流程。

1. 查看注册表

第一步

首先看看项目配置了哪些注册表：

get_project_registries

返回结果：当前项目配置了 @shadcn 和 @react-bits 两个注册表。

2. 浏览可用组件

第二步
list_items_in_registries

@shadcn 注册表下共 405 个组件，包括 button、card、avatar、badge、dropdown-menu、tooltip、kbd 等等。

第五步

踩坑： Windows 上 shadcn 最新版（v4.7+）有路径解析 bug，组件会生成到字面路径 @\components\ui\ 而非 src/components/ui/。降级到 v4.6.0 解决。

7. 审计检查

第七步

  • [x] 所有依赖已安装
  • [x] 检查 lint 和 TypeScript 错误

总结

React Bits MCP 让 shadcn/ui 组件的搜索、预览、安装全部在 Claude Code 对话中完成。7 个 MCP 工具覆盖了从浏览组件库到审计检查的完整流程，不用切换浏览器查文档，不用手动拼 CLI 命令。

核心收益：

  1. 省时 — 不用手动搜组件名、翻文档
  2. 准确 — MCP 返回的命令可直接复制或让 AI 自动执行
  3. 规范 — 审计清单确保组件添加后无遗漏项
