题目十一：哈夫曼编码/译码器

问题描述

利用哈夫曼编码进行信息通信可以大大提高信道利用率，缩短信息传输时间，降低传输成本。对于双工信道，每端都需要一个完整的编/译码系统。设某编码系统共有 n 个字符，使用频率分别为 {w1, w2, …, wn}，设计一个不等长的编码方案，使得该编码系统的空间效率最好。

哈夫曼编码的核心思想：频率高的字符用短编码，频率低的字符用长编码，从而使得加权路径长度最短。

核心算法

构造哈夫曼树

算法步骤
  1. 数组 huffTree 初始化，所有元素的 parent、lchild、rchild 置为 -1
  2. 前 n 个元素的权值置为给定的 w[n]
  3. 进行 n-1 次合并：
    • 在集合中选取两个权值最小且 parent == -1 的根结点 i1, i2
    • 将 i1、i2 合并为一棵新二叉树 k
    • 令 huffTree[i1].parent = k; huffTree[i2].parent = k
    • 左分支设为 0，右分支设为 1

Select 函数——选取最小两权值

编码生成

从叶子结点出发，沿 parent 指针向上回溯至根结点，记录路径方向：

  • 当前结点是双亲的左孩子 → 编码位为 0
  • 当前结点是双亲的右孩子 → 编码位为 1
  • 最后将编码字符串逆序即为该字符的哈夫曼编码

译码

从根结点出发，逐位读取编码串：

  • 读 0 → 走左孩子
  • 读 1 → 走右孩子
  • 到达叶子结点 → 输出对应字符，回到根结点继续

模块划分

函数 功能
CountFrequency 统计输入文本中各字符出现频率
CreateHuffmanTree 根据频率数组构建哈夫曼树
CreateHuffmanCode 从哈夫曼树生成各字符的编码表
Encode 利用编码表对原文进行编码
Decode 利用哈夫曼树对编码串进行译码
PrintCode 打印字符-频率-编码对照表

小结

哈夫曼编码作为一种最优前缀编码方案，在实际通信系统中应用广泛。通过本实验，掌握了以下核心技能：

C 语言中结构体数组的灵活运用

哈夫曼算法的时间复杂度为 O(n²)，主要消耗在每轮合并时查找最小两权值。若采用最小堆优化，可将复杂度降至 O(n log n)。
